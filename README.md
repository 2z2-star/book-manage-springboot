 校园图书管理系统项目介绍
一、项目简介
本项目是基于SpringBoot开发的**校园图书管理后端接口系统**，配套前端可视化操作页面，实现管理员登录、图书全生命周期管理、操作日志实时展示功能，可用于接口测试、课程作业演示。
技术栈：SpringBoot + MyBatis + MySQL + Token登录拦截器

二、核心功能模块
###1. 登录模块（LoginController）
1. 账号密码登录，校验用户身份；
2. 登录成功生成Token令牌，返回前端；
3. Token拦截器全局校验，未登录状态禁止访问所有图书接口。

### 2. 图书管理模块（BookController）
1. **图书查询**：支持书名、作者模糊分页查询，分页展示图书数据；
2. **新增图书**：录入图书实体信息，存入MySQL数据库；
3. **修改图书**：根据图书ID更新书籍信息；
4. **删除图书**：单条图书删除；
5. **批量清空图书**：一键清空全部图书数据；
6. 统一返回封装类 `Result`，标准化接口返回格式。

### 3. 实体与数据库
- `User` 用户表：存储管理员账号信息；
- `Book` 图书表：存储图书名称、作者等业务数据；
- MyBatis XML实现数据库增删改查映射。

### 4. 页面操作日志（前端展示）
页面底部实时打印**操作执行日志**，记录每一次接口请求行为：
模糊搜索、新增、修改、删除、批量清空操作都会生成日志，展示操作时间与操作内容，直观体现接口调用流程。

## 三、接口测试说明
### 1. 前置要求
所有图书相关接口必须携带Token请求头，未携带会被拦截器拦截，返回无权限提示。
登录接口无需Token，优先调用登录获取令牌。

### 2. 核心接口清单
1. 登录接口
- 请求地址：`/login`
- 请求方式：POST
- 请求参数：username、password
- 返回：成功携带Token令牌，用于后续接口鉴权

2. 图书分页模糊查询
- 请求地址：`/book`
- 请求方式：GET
- 参数：page（页码）、size（每页条数）、bookName、author
- 功能：根据关键词模糊检索图书，分页展示，页面日志同步打印搜索记录

3. 新增图书
- 请求地址：`/book`
- 请求方式：POST
- 请求体：Book完整实体信息
- 效果：数据库新增数据，页面日志记录新增操作

4. 修改图书
- 请求地址：`/book`
- 请求方式：PUT
- 请求体：带ID的图书实体
- 效果：根据ID更新书籍信息

5. 删除图书
- 请求地址：`/book/{id}`
- 请求方式：DELETE
- 参数：图书id路径变量

6. 批量清空图书
- 接口内置批量删除逻辑，一键清除全部图书数据

### 3. 测试工具推荐
Postman / IDEA内置HTTP工具 / Apifox均可测试：
1. 先调用登录接口拿到token；
2. 请求图书接口时，请求头添加：`Token: 你的令牌`；
3. 发送请求后，前端页面底部操作日志会实时刷新本次调用记录，直观验证接口是否执行成功。

## 四、项目运行说明
1. 导入项目后配置MySQL连接，修改resources下数据库配置；
2. 执行mapper文件夹SQL映射，创建User、Book数据表；
3. 启动BookserverApplication启动类；
4. 访问前端页面，登录后使用按钮操作，所有操作都会调用后端接口并生成操作日志。

## 五、项目亮点
1. 登录拦截机制，实现接口权限控制；
2. 统一返回结果封装，接口返回格式规范；
3. 前端可视化操作页面，配套实时操作日志，接口测试效果直观；
4. 完整MyBatis CRUD分页逻辑，适合SpringBoot接口实训作业。
